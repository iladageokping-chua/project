<B>ที่มาของปัญหา เเละความสำคัญของโครงงาน</B>

<dd>ในยุคปัจจุบันการออกกำลังกายกลายเป็นส่วนสำคัญในวิถีชีวิตของผู้คน ไม่ว่าจะเป็นการออกกำลังกายเพื่อสุขภาพ</dd> การเสริมสร้างกล้ามเนื้อ หรือการออกกำลังกายเพื่อลดน้ำหนัก การมีข้อมูลและแนวทางที่เหมาะสมจะช่วยให้ทุกคนสามารถบรรลุเป้าหมายของตนได้อย่างมีประสิทธิภาพ อย่างไรก็ตาม หลายคนประสบปัญหาการเริ่มต้นและรักษาวินัยในการออกกำลังกาย บางคนอาจไม่รู้ว่าจะเริ่มต้นอย่างไร หรือไม่สามารถติดตามความก้าวหน้าของตนเองได้อย่างมีประสิทธิภาพ นอกจากนี้ การออกกำลังกายที่ไม่เหมาะสมกับสภาพร่างกายหรือเป้าหมาย อาจนำไปสู่การบาดเจ็บหรือไม่ได้ผลลัพธ์ตามที่ต้องการ
โปรเจคนี้จัดทำขึ้นเพื่อตอบสนองความต้องการของผู้ที่ต้องการติดตามและวางแผนการออกกำลังกายของตนเอง โดยมีเป้าหมายเพื่อช่วยให้ผู้ใช้งานสามารถจัดการกับตารางการฝึกซ้อมได้ดีขึ้น ผ่านการบันทึกข้อมูล และให้คำแนะนำที่เป็นประโยชน์ ระบบจะช่วยให้ผู้ใช้งานสามารถจัดการกับตารางการฝึกซ้อม ติดตามสถิติการออกกำลังกาย ตั้งเป้าหมาย และแนะนำโปรแกรมท่าออกกำลังกายที่เหมาะสม โดยระบบจะวิเคราะห์ข้อมูลและเสนอโปรแกรมการออกกำลังกายที่ตรงกับเป้าหมายของผู้ใช้งาน ไม่ว่าจะเป็นการเพิ่มมวลกล้ามเนื้อ ลดน้ำหนัก หรือพัฒนาความแข็งแกร่งของร่างกาย
นอกจากนี้ ระบบยังช่วยให้ผู้ใช้งานสามารถตรวจสอบความก้าวหน้าของการฝึกซ้อมได้ตลอดเวลา โดยการเก็บข้อมูลการออกกำลังกายอย่างละเอียด และให้คำแนะนำที่เหมาะสมตามสภาพร่างกายและเป้าหมายที่ตั้งไว้ เพื่อให้ผู้ใช้งานทำตามเป้าหมายการออกกำลังกายได้อย่างสะดวกสบาย และไม่มีปัญหาในการรักษาวินัย การมีข้อมูลที่ถูกต้องและเป็นระบบจะช่วยให้ผู้ใช้งานสามารถวางแผนการออกกำลังกายได้อย่างมีประสิทธิภาพ และเพิ่มโอกาสในการประสบความสำเร็จในเรื่องของสุขภาพและฟิตเนส

**จุดประสงค์ของระบบติดตามเป้าหมายการออกกำลังกาย (Workout Tracker)**
1. ช่วยให้ผู้ใช้สามารถติดตามผลลัพธ์ของการออกกำลังกายได้อย่างเป็นระบบ - ระบบช่วยให้ผู้ใช้สามารถดูพัฒนาการของตนเองได้ตลอดเวลา ไม่ว่าจะเป็นระยะเวลาที่ใช้ในการออกกำลังกาย จำนวนครั้งที่ฝึกฝน หรือความแข็งแรงของร่างกายที่เพิ่มขึ้น
2. กระตุ้นให้เกิดความสม่ำเสมอ – การตั้งเป้าหมายที่ชัดเจนช่วยให้ผู้ใช้มีแรงจูงใจในการออกกำลังกายอย่างต่อเนื่อง ลดปัญหาการหยุดออกกำลังกายกลางคัน
3. วิเคราะห์ข้อมูล – ระบบสามารถช่วยให้ผู้ใช้สามารถวางแผนการออกกำลัง และเป้าหมายของตนเองได้
4. ลดความเสี่ยงจากการออกกำลังกายที่ไม่เหมาะสม – ผู้ใช้สามารถเลือกโปรแกรมออกกำลังกายที่เหมาะสมกับร่างกาย และสภาวะสุขภาพของตนเอง ลดความเสี่ยงต่ออาการบาดเจ็บจากการออกกำลังกายที่หนักเกินไป หรือไม่ถูกต้อง
5. ช่วยบริหารเวลาให้มีประสิทธิภาพ – ระบบช่วยให้ผู้ใช้สามารถวางแผนการออกกำลังกายให้สอดคล้องกับไลฟ์สไตล์และตารางชีวิตประจำวัน ทำให้สามารถออกกำลังกายได้โดยไม่กระทบต่อหน้าที่อื่นๆ
6. เพิ่มความสะดวกสบายในการจัดการตารางการฝึกซ้อม – ผู้ใช้สามารถกำหนดเวลาออกกำลังกาย เลือกประเภทของกิจกรรม และปรับเปลี่ยนตารางได้ตามความต้องการ
7. ส่งเสริมสุขภาพที่ดีและลดความเสี่ยงของโรคต่างๆ - เป็นการดึงดูดหันคนหันมาสนใจการออกกำลังกายมากขึ้น โดยที่ผู้ใช้สามารถใช้คำแนะนำจากระบบในการช่วยวางแผนการออกกำลังกาย
8. เพิ่มความสะดวกในการเข้าถึงข้อมูล และแหล่งความรู้เกี่ยวกับการออกกำลังกาย - ผู้ใช้สามารถทราบถึงแผนการออกกำลังกายของตนเอง และกำหนดแผนการเองได้
9. ส่งเสริมสุขภาพที่ดี และลดความเสี่ยงของโรคต่าง ๆ - เนื่องจากเป็นระบบติดตามการออกกำลังกาย เมื่อออกกำลังกายได้ตามเป้าหมาายที่ตั้งไว้จะส่งผลดีต่อสุขภาพของตัวผู้ใช้เอง

**ประโยชน์ของระบบติดตามเป้าหมายการออกกำลังกาย (Workout Tracker)**
1. ติดตามผลและพัฒนาการของการออกกำลังกาย  
• ผู้ใช้สามารถดูสถิติย้อนหลังของตนเอง เช่น จำนวนก้าวที่เดิน เวลาที่ใช้ในการออกกำลังกาย หรือระดับความแข็งแรงที่เพิ่มขึ้น \
• ทำให้สามารถวิเคราะห์แนวโน้มและปรับแผนการออกกำลังกายให้เหมาะสมกับเป้าหมายของตนเอง
2. ช่วยกระตุ้นให้เกิดความสม่ำเสมอในการออกกำลังกาย\
• การตั้งเป้าหมาย และบันทึกความก้าวหน้าทำให้ผู้ใช้มีแรงจูงใจในการออกกำลังกายต่อเนื่อง\
• ช่วยลดปัญหาการหยุดออกกำลังกายกลางคันเนื่องจากขาดแรงกระตุ้น
3. วางแผน และบริหารเวลาการออกกำลังกายได้ง่ายขึ้น\
• ผู้ใช้สามารถกำหนดตารางเวลาการออกกำลังกายล่วงหน้า และเลือกรูปแบบการฝึกที่เหมาะสมกับตนเอง\
• ลดปัญหาการออกกำลังกายแบบไม่มีแบบแผน หรือขาดความต่อเนื่อง
4. ช่วยให้การออกกำลังกายมีประสิทธิภาพมากยิ่งขึ้น\
• ระบบสามารถแนะนำรูปแบบการออกกำลังกายที่เหมาะสมกับเป้าหมายของผู้ใช้ เช่น การลดน้ำหนัก การเพิ่มกล้ามเนื้อ หรือการพัฒนาความแข็งแรง\
• ป้องกันการออกกำลังกายที่ไม่ถูกต้องหรือหนักเกินไป ซึ่งอาจทำให้เกิดการบาดเจ็บ
5. สนับสนุนการดูแลสุขภาพในระยะยาว\
• ทำให้ผู้ใช้สามารถควบคุมสุขภาพของตนเองได้ดีขึ้น ลดความเสี่ยงต่อโรคต่างๆ เช่น โรคหัวใจ เบาหวาน และโรคอ้วน\
• เสริมสร้างสมรรถภาพร่างกาย ทำให้มีพลังงานในการใช้ชีวิตประจำวันมากขึ้น
6. ลดความยุ่งยากในการจดจำ และบันทึกข้อมูลด้วยตนเอง\
• ระบบช่วยบันทึกข้อมูลโดยอัตโนมัติ ทำให้ผู้ใช้ไม่ต้องจดจำรายละเอียดทั้งหมดด้วยตัวเอง\
• สามารถเรียกดูข้อมูลย้อนหลังได้สะดวก ทำให้วิเคราะห์ และปรับแผนการออกกำลังกายได้ง่ายขึ้น

**ขอบเขตของโครงงาน** 
- เพื่อให้ผู้ใช้สามารถกำหนดเป้าหมายการออกกำลังกายของตนเองได้อย่างอิสระและยืดหยุ่น
- เพื่อให้ผู้ใช้สามารถบันทึกและติดตามความก้าวหน้าในการออกกำลังกายของตนเองได้อย่างเป็นระบบ
- เพื่อช่วยให้ผู้ใช้ได้รับคำแนะนำในการออกกำลังกายที่เหมาะสมกับเป้าหมายเพิ่มประสิทธิภาพในการออกกำลังกาย และป้องกันการบาดเจ็บ
- เพื่อให้ผู้ใช้ทุกคนสามารถใช้งานระบบได้ง่าย สะดวก และมีประสิทธิภาพ

**Functional Requirement and Non-Functional Requirement**\
**1. Functional Requirements (ความต้องการเชิงฟังก์ชัน)**

ฟังก์ชันหลักที่ต้องมีในระบบ ได้แก่

1.1 การลงทะเบียนและจัดการบัญชีผู้ใช้

• ผู้ใช้สามารถ สมัครสมาชิก และ เข้าสู่ระบบ ได้

• อายุ, เพศ, น้ำหนัก, ส่วนสูง

• เป้าหมายการออกกำลังกาย (ลดน้ำหนัก, เพิ่มกล้ามเนื้อ, ฟิตเนสทั่วไป ฯลฯ)

1.2 ระบบตั้งเป้าหมายการออกกำลังกาย

• ผู้ใช้สามารถกำหนด เป้าหมาย ได้ เช่น ระยะเวลาการออกกำลังกายต่อวัน/สัปดาห์

• ปริมาณแคลอรีที่ต้องการเผาผลาญ

1.3 ระบบบันทึกและติดตามการออกกำลังกาย

• ผู้ใช้สามารถบันทึกกิจกรรมออกกำลังกาย เช่น เดิน, วิ่ง, ปั่นจักรยาน, ว่ายน้ำ, โยคะ, เวทเทรนนิ่ง ฯลฯ

• แสดงสถิติ เช่น เวลาที่ใช้, ระยะทาง/จำนวนก้าว, ปริมาณแคลอรีที่เผาผลาญ

1.4 ระบบแนะนำโปรแกรมออกกำลังกาย

• แนะนำโปรแกรมออกกำลังกายตาม เป้าหมายของผู้ใช้

• สามารถเลือกแผนฝึกซ้อม เช่น เวทเทรนนิ่ง, คาร์ดิโอ, HIIT, โยคะ

1.5 ระบบวิเคราะห์ผลและแสดงสถิติ

• แสดงข้อมูลเป็น Dashboard เช่น กราฟแสดงแคลอรีที่เผาผลาญ

• แสดงผลลัพธ์รายการที่ทำไปในแต่ละวัน, เดือน, ปี

**2. Non-Functional Requirements (ความต้องการเชิงคุณภาพ)**

2.1 ความปลอดภัยของข้อมูล (Security)

• เข้ารหัสข้อมูล ผู้ใช้เพื่อป้องกันการรั่วไหล

• ป้องกัน SQL Injection, XSS, CSRF

2.2 ประสิทธิภาพของระบบ (Performance & Scalability)

• ต้องรองรับ ผู้ใช้จำนวนมาก โดยไม่ทำให้ระบบล่ม

• ควรโหลดข้อมูลได้อย่างรวดเร็ว

2.3 การรองรับอุปกรณ์และแพลตฟอร์ม (Compatibility & Accessibility)

• รองรับ iOS และ Android

• รองรับการทำงานบน Web App

2.4 การใช้งานง่าย (Usability & UX/UI)

• interface ใช้งานง่าย และไม่ซับซ้อน

• ออกแบบ UI ให้ ดูสะอาดตา และใช้งานสะดวก

2.5 ความสามารถในการอัปเดตและขยายระบบ (Maintainability & Extensibility)

• ระบบต้องสามารถ อัปเดตฟีเจอร์ใหม่ ได้โดยไม่กระทบการทำงานหลัก

• โครงสร้างโค้ดต้องสามารถ ปรับปรุงและขยายได้ง่าย

**Process**

*   ศึกษา สำรวจพฤติกรรมความต้องการของผู้ใช้ และฟังก์ชันที่จำเป็นต้องมี เป้าหมายของผู้ใช้โดยวิเคราะห์ความต้องการเฉพาะ 
*   ออกแบบ UI/UX เพื่อกำหนดโครงสร้างเว็บ และการทำงานของฟังก์ชันต่างๆ เพื่อนำมาพัฒนา และทดสอบการทำงานของระบบ  
    
*   จัดทำรายงาน และเอกสารที่เกี่ยวข้องกับการทำโปรเจค บันทึกข้อผิดพลาด แนวทางการแก้ไข
*   ทบทวน วิเคราะห์ปัญหาที่เกิดขึ้นระหว่างการพัฒนาระบบ และนำมาปรับปรุงการทำงานของกลุ่ม

**Method**

*   การเก็บรวบรวม และการวิเคราะห์ความต้องการ
*   การออกแบบ UI/UX เพื่อกำหนดโครงสร้าง และการทำงานของระบบ
*   การพัฒนา โดยใน Frontend จะพัฒนา UI และฟีเจอร์ต่างๆ ส่วนใน Backend จะประมวลผลข้อมูล และทำการ Coding
*   การทดสอบระบบ ทดสอบ UI/UX โดยให้ผู้ใช้กลุ่มทดลองใช้งาน และเก็บ Feedback
*   การปรับปรุง เมื่อรับ Feedback มาแล้วทำการวิเคราะห์ถึงความผิดพลาดที่เกิดขึ้น และนำมาพัฒนาต่อในอนาคต

**Tools**

*   Visual Studio Code ใช้สำหรับเขียนโค้ด
*   GitHub, GitLab ใช้จัดการ version ของโค้ด
*   Figma ใช้สำหรับออกแบบ UI
*   Microsoft Azure ใช้สำหรับการจัดทำรายงาน
*   Google Drive ใช้ส่งไฟล์ข้อมูล รายงาน โค้ด
*   Retrospective โดยใช้ Discord เพื่อประชุมออนไลน์

**สรุปการประชุม
ฟังก์ชันที่จะทำในโปรเจคนี้ ได้แก่**
- ผู้ใช้สามารถเพิ่มกิจกรรมได้ เช่น ว่ายน้ำ 30 นาที, โยคะ
- กำหนดวันเวลาของกิจกรรม เช่น ต้องทำทุกวัน, วีคละ 2 ครั้ง
- ค้นหากิจกรรมที่ต้องทำ
- Checklist ว่ากิจกรรมไหนทำแล้วบ้าง
- ตั้งเป้าหมายของการออกกำลังกาย
- มีการแนะนำโปรแกรมการออกกำลังกาย หรือข้อมูล
- โหมดเล่นเพลงประกอบ (ถ้าหากมีเวลาเหลือ และสามารถทำได้)
